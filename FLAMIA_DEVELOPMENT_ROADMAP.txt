1. ROADMAP OVERVIEW
================================================================================

  Phase  Name                  Duration    Dependencies     Priority
  ─────  ────────────────────  ──────────  ───────────────  ────────
  1      Database Setup        3 days      None             Critical
  2      Backend Core          5 days      Phase 1          Critical
  3      Security              4 days      Phase 2          Critical
  4      Product Module        5 days      Phase 3          Critical
  5      Order Module          6 days      Phase 4          Critical
  6      Payment Integration   5 days      Phase 5          Critical
  7      Review System         4 days      Phase 5          High
  8      Frontend Static UI    10 days     Phase 1 (schema) Critical
  9      API Integration       8 days      Phases 4-7, 8    Critical
  10     Deployment            4 days      Phases 1-9       Critical

  Total Estimated Duration: 54 working days (approximately 11 weeks)

  Parallel Execution Opportunities:
    - Phase 8 (Frontend Static UI) can begin in parallel with Phases 4-7
      since it starts with static/mock data.
    - Phase 7 (Reviews) can run in parallel with Phase 6 (Payments)
      since they share only the Order dependency.

  With parallelization, estimated timeline reduces to approximately
  8-9 weeks with two developers (one backend, one frontend).

2. PHASE 1 — DATABASE SETUP (3 DAYS)
================================================================================

  Objective:
    Set up Supabase project, configure PostgreSQL database, create all
    tables, indexes, and storage buckets.

  Prerequisites:
    - Supabase account created
    - Supabase project provisioned

  Deliverables:
    ☐ Supabase project configured with proper region (Mumbai/Singapore)
    ☐ All 12 database tables created with proper constraints
    ☐ All indexes created (including pg_trgm extension)
    ☐ All foreign keys and cascade rules configured
    ☐ Storage buckets created (flamia-products, flamia-reviews, flamia-payments)
    ☐ Storage bucket RLS policies configured
    ☐ Initial category seed data inserted
    ☐ Database connection tested from local development environment
    ☐ SQL migration scripts saved in repository

  Day-by-Day Breakdown:
    Day 1: Supabase setup, core tables (users, categories, products,
            product_images)
    Day 2: Transaction tables (orders, order_items, payments, addresses,
            coupons, coupon_usages)
    Day 3: Review tables, indexes, storage buckets, seed data, testing

  Verification:
    - All tables visible in Supabase Table Editor
    - Constraints verified (try inserting invalid data)
    - Storage buckets accessible via Supabase dashboard
    - Connection string works from local Spring Boot application

4. PHASE 3 — SECURITY (4 DAYS)
================================================================================

  Objective:
    Implement Supabase JWT validation, Spring Security configuration,
    role-based access control, and CORS configuration.

  Prerequisites:
    - Phase 2 complete (entities and repositories ready)
    - Supabase JWT secret available

  Deliverables:
    ☐ JwtTokenValidator — validates Supabase JWT (signature, expiry, claims)
    ☐ JwtAuthenticationFilter — extracts and validates JWT from requests
    ☐ UserPrincipal class for authenticated context
    ☐ SecurityConfig — Spring Security filter chain configuration
    ☐ CorsConfig — CORS rules for frontend origin
    ☐ Public endpoint whitelist configured
    ☐ ADMIN role enforcement on /api/admin/** paths
    ☐ AuthController — POST /api/auth/sync endpoint
    ☐ AuthService — user sync logic (create/update on first login)
    ☐ Security headers configured
    ☐ All endpoints tested with valid/invalid/missing JWT
    ☐ Role-based access verified (CUSTOMER vs ADMIN)

  Day-by-Day Breakdown:
    Day 1: JwtTokenValidator, JwtAuthenticationFilter, UserPrincipal
    Day 2: SecurityConfig, CorsConfig, public endpoint whitelist
    Day 3: AuthController, AuthService, user sync flow
    Day 4: Integration testing — JWT validation, role checks, CORS

  Verification:
    - Public endpoints accessible without JWT
    - Protected endpoints return 401 without JWT
    - Valid JWT with CUSTOMER role can access customer endpoints
    - Valid JWT with CUSTOMER role gets 403 on admin endpoints
    - Valid JWT with ADMIN role can access admin endpoints
    - CORS headers present on responses
    - Invalid JWT returns 401

13. RISK MITIGATION
================================================================================

  Risk                              Mitigation
  ──────────────────────────────── ────────────────────────────────────────
  Render free tier cold starts     Document expected behavior for users.
  (30-second spin-up delay)        Show loading state on frontend.
                                   Consider keep-alive ping from external
                                   service (e.g., cron-job.org).

  Supabase free tier limits        Monitor row counts and storage usage.
  (500MB database, 1GB storage)    Optimize images before upload.
                                   Clean up unused media files.

  Razorpay integration issues      Start with Razorpay test mode.
                                   Manually test all payment scenarios.
                                   UPI fallback provides redundancy.

  SMS OTP delivery failures        Display retry option with clear
                                   messaging. Log failures for debugging.
                                   Consider fallback OTP channel (WhatsApp)
                                   in future iteration.

  Large image uploads on slow      Client-side compression before upload.
  mobile connections               Progress indicator during upload.
                                   Max 3MB limit already constrains size.

  Scope creep during development   Stick to documented features only.
                                   Defer enhancements to post-launch phase.
                                   Track all change requests separately.
